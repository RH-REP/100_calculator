# ゲーム3 工程表

仕様書「数の開拓村」に沿って、ゲーム3は既存版に依存しない新規プロトタイプとして進める。

## 前提

- 仕様書のMVP範囲は、1〜20の加減算、5・10のまとまり、探索、建築、製作、決定論的なターン制戦闘。
- 今回はレビュー用GUIのみを作成し、数量操作、判定、保存、戦闘処理、ミッション進行は実装しない。
- 表現はトップダウン2Dのピクセル風UIを仮採用する。

## フェーズ0.1：GUIレビュー版

成果物：

- `game3.html`
- 村、ミッション、探索準備、荷造り、建築、工房、戦闘、結果、設定系の主要画面
- 5個列、10箱、ばら素材、投入枠、HP、残量プレビュー、式再生の見た目
- タップ移動、ヒント、動きを減らす、音声読み上げなどの設定UI

レビュー観点：

- 子どもが素材の種類と個数を見分けられるか
- 10箱と5個列が同じ規則で読めるか
- 建築・荷造り・戦闘で、次に見るべき数が明確か
- 否定的な誤答表示ではなく「あと何個」「何個多い」という表現になっているか
- 横画面1280×720相当で主要ボタンが押しやすいか

完了条件：

- 既存ゲームファイルを参照せず、ゲーム3の新規HTMLとして開ける
- GUI上で主要画面のレビューができる
- 機能未実装であることがコード上でも明確に分離されている
